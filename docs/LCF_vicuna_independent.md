# Paper B (LCF) — Independent Cross-Check on Vicuna-7b

A **second, independent** from-scratch reproduction of LCF, built as a separate
codebase (`lcf/independent_vicuna/`) and run on **lmsys/vicuna-7b-v1.5** — one of
the paper's own six models. Its purpose is to cross-check the team's Qwen3-8B
result with a different implementation, a different model, and a different judge.
**Headline: on Vicuna, LCF does not help on any metric** — including the ΔProb
effect that reproduced on Qwen3. The two runs together show LCF's one reproducible
effect is model-dependent, and the paper's headline gains reproduce on neither.

## Setup actually run
- Same LFUD scenario split 540/60/204; 804 valid conclusions generated by
  **Claude-Sonnet-4.6** (this codebase had an Anthropic key, not OpenAI), 804
  invalid = original LFUD fallacious conclusions.
- Vicuna-7b-v1.5 in **4-bit (NF4)** on a single H100 slice. LCF on layers 10–30
  (attn+MLP), AdamW, 15 epochs. **lr = 1e-4 + grad-clip 1.0** — the paper's 1e-3
  **diverged** here (reconstruction loss spiked >80 with the cross-attention
  decoder); τ = 0.2.
- Eval: Conclusion Generation (η=0.5) judged by **Claude-Sonnet-4.6** + greedy-text
  perplexity; Fallacy Identification 4-option (η=4.5) via length-normalized option
  log-probs → Accuracy, ΔProb. Steering magnitude η swept 0.25–8.0.

## Results (Vicuna-7b, test set)

| Variant | Valid%(judge)↑ | PPL↓ | Acc↑ | ΔProb (×100)↑ |
|---|---|---|---|---|
| original | 36.2 | 1.49 | 69.6 | 34.4 |
| **+LCF (η=4.5 id / 0.5 gen)** | 31–35 | 1.50–2.98 | **63.7** | **30.0** |

*Valid% by Claude judge (n=80); Acc/ΔProb on the 204-item test set. ΔProb shown ×100
to match the paper/team convention (raw 0.344 → 0.300).*

## Findings (honest)
1. **No metric improves.** Identification accuracy 69.6 → 63.7 and ΔProb 34.4 → 30.0
   both **drop**; an η sweep (0.25–8.0) shows LCF is neutral for η≤2 and harmful
   beyond (Acc 71.7 → 68.3 at η=4.5, → 65.8 at η=8.0). Generation Valid% is flat-to-down
   (36.2 → 31–35) and perplexity degrades sharply at η=4.5 (1.49 → 2.98).
2. **The disentanglement never forms (root cause).** t-SNE of validation reps shows
   valid/invalid **intermixed in both the content and the logic space**
   (`lcf/independent_vicuna/results/figures/tsne_content_logic.png`); per-tap
   nearest-centroid separability peaks at **0.66** (chance 0.50) and the contrastive
   loss stays flat. With no separable direction the steering vector is near-noise.
3. **Controlled experiments rule out two confounds.** fp16 instead of 4-bit barely
   moves separability (0.66 → 0.67); regenerating the valid conclusions with Claude
   (higher quality, correct hedging) **lowers** it to 0.57 — minimal-edit pairs are
   textually closer and thus *harder* to separate. Neither recovers the mechanism.
4. **Module is correct.** A synthetic CPU unit test (validity on a known axis) shows
   the module disentangles, separates (within-class cos ≈ 1.0 / across ≈ −0.78), and
   steers bidirectionally as designed — so this is an empirical failure of the
   separability *assumption*, not a code bug.

## Relation to the team's Qwen3-8B result
Not a contradiction — a **generalization**. The Qwen3 run reproduced exactly one
effect (ΔProb ≈2×) while accuracy and generation were flat; the paper's large
accuracy/generation gains reproduced there either. On Vicuna even the ΔProb effect
disappears. Joint reading: **LCF's only reproducible signal (ΔProb) is
model-dependent, and the paper's headline improvements reproduce on no model we
tested** — consistent with a fragile separability assumption.

## Differences from the team pipeline (why numbers aren't directly comparable)
- Separate codebase (`lcf/independent_vicuna/`), not `lcf/lcf_impl/`.
- Judge = Claude-Sonnet-4.6 (team: GPT-4o); no trained discriminator (no ValidTrained).
- Our 4-option identification set makes the valid choice easier → baseline Acc 69.6 /
  ΔProb 0.344 are much higher than the team's Qwen3 baseline (31.9 / 0.040), leaving
  little headroom for LCF. This is itself evidence that the paper under-specifies the
  identification protocol.
- lr 1e-4 + grad-clip (team used 1e-3); τ=0.2 (team 0.1); 4-bit (team bf16-ish).
